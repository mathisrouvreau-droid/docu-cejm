Configuration DNS

DNS (Domain Name System) traduit les noms de domaine (www.exemple.com) en adresses IP (192.168.1.100). Indispensable pour la navigation web et le fonctionnement d'Active Directory.

Packet Tracer : serveur DNS

! Sur le serveur DNS (onglet Services > DNS) :
! 1. Activer le service DNS (ON)
! 2. Ajouter les enregistrements :
!    Type A : www.exemple.com  → 192.168.1.100
!    Type A : ftp.exemple.com  → 192.168.1.101
!    Type A : mail.exemple.com → 192.168.1.102

DNS sur routeur Cisco

! Activer la résolution DNS
ip domain-lookup
ip name-server 192.168.1.2

! Créer des entrées DNS statiques
ip host serveur-web 192.168.1.100
ip host serveur-ftp 192.168.1.101

Types d'enregistrements DNS

TypeDescriptionExemple
ANom → IPv4www.exemple.com → 192.168.1.100
AAAANom → IPv6www.exemple.com → 2001:db8::1
CNAMEAlias vers un autre nomweb.exemple.com → www.exemple.com
MXServeur de messagerieexemple.com → mail.exemple.com
NSServeur DNS autoritaireexemple.com → ns1.exemple.com
PTRIPv4 → Nom (reverse)192.168.1.100 → www.exemple.com
SOADébut d'autorité de zoneInfos sur la zone DNS

Configuration DHCP

DHCP (Dynamic Host Configuration Protocol) attribue automatiquement les paramètres IP aux clients : adresse IP, masque, passerelle, DNS. Processus : DORA (Discover, Offer, Request, Acknowledge).

DHCP sur routeur Cisco

! Exclure les adresses réservées (serveurs, routeur, etc.)
ip dhcp excluded-address 192.168.1.1 192.168.1.20

! Créer le pool DHCP
ip dhcp pool LAN_COMPTA
 network 192.168.1.0 255.255.255.0
 default-router 192.168.1.1
 dns-server 192.168.1.2
 domain-name exemple.local
 lease 7

Pool DHCP pour un second VLAN

ip dhcp excluded-address 192.168.2.1 192.168.2.20

ip dhcp pool LAN_IT
 network 192.168.2.0 255.255.255.0
 default-router 192.168.2.1
 dns-server 192.168.1.2

Relay DHCP (ip helper-address)

! Si le serveur DHCP est sur un autre réseau/VLAN,
! configurer le relay sur l'interface du routeur côté client :
interface GigabitEthernet0/0
 ip helper-address 192.168.10.100

ip helper-address : relaye les broadcasts DHCP (et autres) vers l'adresse unicast du serveur DHCP situé sur un autre réseau. Indispensable quand le serveur DHCP n'est pas sur le même sous-réseau que les clients.

Packet Tracer : serveur DHCP

! Sur le serveur (onglet Services > DHCP) :
! 1. Activer le service DHCP (ON)
! 2. Configurer le pool :
!    Pool Name     : serverPool
!    Default Gateway : 192.168.1.1
!    DNS Server    : 192.168.1.2
!    Start IP      : 192.168.1.21
!    Subnet Mask   : 255.255.255.0
!    Max Users     : 200

Vérification DHCP

! Voir les baux DHCP attribués
show ip dhcp binding

! Voir les conflits d'adresses
show ip dhcp conflict

! Statistiques DHCP
show ip dhcp pool

! Sur le PC client
ipconfig /release
ipconfig /renew
ipconfig /all

Active Directory (simulé)

Active Directory (AD) est le service d'annuaire de Microsoft. Il centralise la gestion des utilisateurs, groupes, ordinateurs et stratégies (GPO) dans un domaine. AD repose sur DNS (obligatoire), LDAP (annuaire) et Kerberos (authentification).

Composants clés d'AD

ComposantDescription
DomaineUnité de base d'AD (ex : exemple.local). Regroupe les objets gérés.
Contrôleur de domaine (DC)Serveur hébergeant la base AD et gérant l'authentification.
OU (Organizational Unit)Conteneur pour organiser les objets (utilisateurs, PC) et appliquer des GPO.
GPO (Group Policy Object)Stratégie de groupe appliquée aux utilisateurs/ordinateurs d'une OU.
ForêtEnsemble de domaines partageant un schéma et un catalogue global.
ArbreHiérarchie de domaines partageant un espace de noms contigu.
Catalogue globalRéplique partielle de tous les objets de la forêt (recherches rapides).

Packet Tracer : simulation AD

! Dans Packet Tracer, AD n'est pas directement simulable.
! On simule les services associés :
!
! 1. Serveur DNS : résolution du domaine (exemple.local)
!    Enregistrement A : dc1.exemple.local → 192.168.1.10
!
! 2. Serveur DHCP : distribution des paramètres réseau
!    DNS server = IP du contrôleur de domaine
!    Domain-name = exemple.local
!
! 3. Sur les PCs : configurer le DNS vers le DC
!    ipconfig /all → vérifier DNS = 192.168.1.10
!    nslookup dc1.exemple.local

Configuration NTP

NTP (Network Time Protocol) synchronise l'horloge des équipements réseau avec un serveur de temps. Essentiel pour les logs, les certificats, Kerberos (AD) et la corrélation d'événements de sécurité.

Packet Tracer : serveur NTP

! Sur le serveur (onglet Services > NTP) :
! 1. Activer le service NTP (ON)
! 2. Authentification optionnelle

NTP sur routeur / switch Cisco

! Configurer le serveur NTP
ntp server 192.168.1.3

! Configurer le fuseau horaire
clock timezone CET 1
clock summer-time CEST recurring

! Vérification
show ntp status
show ntp associations
show clock

Stratum : niveau hiérarchique NTP. Stratum 0 = horloge atomique. Stratum 1 = serveur directement connecté. Un routeur configuré comme client NTP aura typiquement un stratum de N+1 par rapport à son serveur.

Schéma réseau type

Architecture classique d'examen

                    Internet
                       |
                  [R1 - Routeur]
                  g0/1: 203.0.113.1 (WAN)
                  g0/0: lien trunk vers SW1
                  g0/0.10: 192.168.10.1 (VLAN 10)
                  g0/0.20: 192.168.20.1 (VLAN 20)
                       |
                  [SW1 - Switch L2]
                  g0/1: trunk vers R1
                  fa0/1-10: access vlan 10
                  fa0/11-20: access vlan 20
                  fa0/24: access vlan 10 (serveurs)
                 /    |    \
              [PC1]  [PC2]  [Serveurs]
          VLAN 10  VLAN 20   VLAN 10

ping 192.168.10.2

show ip route              ! Vérifier les routes

show ip nat translations   ! Vérifier NAT

Points clés à retenir
  • DNS est obligatoire pour Active Directory et la navigation web
  • DHCP : processus DORA (Discover, Offer, Request, Acknowledge)
  • ip helper-address relaye les broadcasts DHCP vers un serveur distant (relay)
  • Les serveurs ont des IP statiques (exclues du pool DHCP)
  • Active Directory = DNS + LDAP + Kerberos. Nécessite un DC et un DNS fonctionnel
  • NTP synchronise l'heure. Indispensable pour Kerberos (tolérance de 5 min), les logs et les certificats
  • Connaître les ports standards : DNS=53, DHCP=67/68, HTTP=80, HTTPS=443, FTP=21, SSH=22, NTP=123
  • Toujours tester la connectivité de proche en proche : ping GW → ping serveur → ping inter-VLAN → ping Internet
  • Vérifier nslookup pour confirmer que le DNS fonctionne
  • Un ip helper-address par interface/VLAN où il y a des clients DHCP
