MÉMO RAPIDE

Mémo Rapide — Packet Tracer

Toutes les commandes essentielles en un coup d'oeil

1
NAT / PAT

NAT Statique (1 IP priv ↔ 1 IP pub)

int g0/0
 ip nat inside
int g0/1
 ip nat outside

ip nat inside source static 192.168.1.10 203.0.113.10

PAT / Overload (N IP priv → 1 IP pub)

access-list 1 permit 192.168.1.0 0.0.0.255

ip nat inside source list 1 interface g0/1 overload

NAT Dynamique (pool)

ip nat pool MON_POOL 203.0.113.1 203.0.113.14 netmask 255.255.255.240
access-list 1 permit 192.168.1.0 0.0.0.255

ip nat inside source list 1 pool MON_POOL

Vérification

show ip nat translations
show ip nat statistics
debug ip nat
2
ACL (Access Control Lists)

ACL Standard numérotée (1-99)

access-list 10 permit 192.168.1.0 0.0.0.255
access-list 10 deny any

int g0/0
 ip access-group 10 in

ACL Étendue numérotée (100-199)

access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 80
access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 443
access-list 100 deny ip any any

int g0/0
 ip access-group 100 in

ACL Nommée

ip access-list standard AUTORISER_LAN
 permit 192.168.1.0 0.0.0.255
 deny any

ip access-list extended WEB_ONLY
 permit tcp any any eq 80
 permit tcp any any eq 443
 deny ip any any
3
OSPF

Configuration de base

router ospf 1
 router-id 1.1.1.1
 network 192.168.1.0 0.0.0.255 area 0
 network 10.0.0.0 0.0.0.3 area 0
 passive-interface g0/0

Route par défaut

ip route 0.0.0.0 0.0.0.0 <IP-FAI>
router ospf 1
 default-information originate

Vérification

show ip ospf neighbor
show ip ospf interface brief
show ip route ospf
show ip ospf database

RIP

Configuration RIPv2

router rip
 version 2
 no auto-summary
 network 192.168.1.0

 passive-interface g0/0
 default-information originate

Vérification

show ip rip database
show ip route rip

7
Règles de placement ACL

ACL Standard : placer au plus proche de la destination (car ne filtre que l'adresse source).

ACL Étendue : placer au plus proche de la source (car filtre source, destination, protocole, port).

Rappel : il y a un deny any implicite en fin de toute ACL. Pensez à ajouter un permit si nécessaire. Une seule ACL par interface, par direction (in/out).

10
Check-list avant de soumettre

Vérifications finales

  • ip nat inside / outside sur les bonnes interfaces ?
  • ACL appliquée sur la bonne interface et dans la bonne direction (in/out) ?
  • network OSPF/RIP avec les bons wildcards / réseaux ?
  • passive-interface sur les interfaces LAN ?
  • no shutdown sur toutes les interfaces utilisées ?
  • switchport mode access/trunk correctement configuré ?
  • encapsulation dot1Q sur les sous-interfaces router-on-a-stick ?
  • ip helper-address configuré pour le relay DHCP ?
  • default-information originate pour redistribuer la route par défaut ?
  • Ping de bout en bout fonctionne (PC → serveur) ?
  • DNS résolu (nslookup / navigateur web) ?
  • show ip route : toutes les routes sont présentes ?
