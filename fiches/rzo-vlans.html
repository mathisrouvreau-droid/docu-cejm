SISR RZO

VLANs (Virtual LAN)

Segmentation réseau, ports Access/Trunk, 802.1Q, inter-VLAN routing

1
Concepts fondamentaux

Un VLAN (Virtual LAN) est un réseau local virtuel qui segmente logiquement un switch physique en plusieurs domaines de broadcast distincts. Les hôtes d'un VLAN ne peuvent pas communiquer avec ceux d'un autre VLAN sans routage (couche 3).

ConceptDescription
VLAN 1VLAN par défaut (natif). Tous les ports y sont assignés initialement. Ne pas l'utiliser en production.
Port AccessPort appartenant à un seul VLAN. Connecte un hôte (PC, serveur, imprimante).
Port TrunkPort transportant le trafic de plusieurs VLANs. Connecte switch-switch ou switch-routeur.
802.1QProtocole d'encapsulation trunk. Ajoute un tag de 4 octets (dont VLAN ID) dans la trame Ethernet.
VLAN natifVLAN dont le trafic n'est pas tagué sur un trunk (par défaut VLAN 1). Doit être identique des deux côtés.
VTPVLAN Trunking Protocol : synchronise la base VLAN entre switches. Modes : Server, Client, Transparent.

3
Configuration des ports Access
! Assigner des ports au VLAN 10 (COMPTA)
interface range FastEthernet0/1-10
 switchport mode access
 switchport access vlan 10

! Assigner des ports au VLAN 20 (IT)
interface range FastEthernet0/11-20
 switchport mode access
 switchport access vlan 20

! Assigner des ports au VLAN 30 (DIRECTION)
interface range FastEthernet0/21-24
 switchport mode access
 switchport access vlan 30

switchport mode access : force le port en mode access (un seul VLAN).
switchport access vlan X : assigne le port au VLAN X.

4
Configuration des ports Trunk
! Configurer un trunk entre deux switches
interface GigabitEthernet0/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,99
 switchport trunk native vlan 99

switchport trunk encapsulation dot1q : utilise 802.1Q (obligatoire sur certains switches).
switchport mode trunk : force le port en mode trunk.
allowed vlan : limite les VLANs autorisés sur le trunk (sécurité).
native vlan 99 : change le VLAN natif (éviter VLAN 1 pour la sécurité).

Important : le VLAN natif doit être identique des deux côtés du trunk, sinon le switch affiche un « Native VLAN mismatch » et le trafic non tagué sera mal acheminé.

5
Inter-VLAN : Router-on-a-stick

Router-on-a-stick : un routeur connecté au switch via un seul lien trunk. Des sous-interfaces sont créées sur le routeur, une par VLAN, pour assurer le routage inter-VLAN.

Configuration du routeur

interface GigabitEthernet0/0
 no shutdown

interface GigabitEthernet0/0.10
 description VLAN 10 - COMPTA
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0

interface GigabitEthernet0/0.20
 description VLAN 20 - IT
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0

interface GigabitEthernet0/0.30
 description VLAN 30 - DIRECTION
 encapsulation dot1Q 30
 ip address 192.168.30.1 255.255.255.0

interface GigabitEthernet0/0.99
 description VLAN 99 - GESTION (natif)
 encapsulation dot1Q 99 native
 ip address 192.168.99.1 255.255.255.0

N'oubliez pas : l'interface physique (g0/0) doit être activée avec no shutdown, et le port du switch connecté au routeur doit être en mode trunk.

Configuration du switch (port vers le routeur)

interface GigabitEthernet0/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,99
 switchport trunk native vlan 99
6
Inter-VLAN : Switch L3 / SVI

Un Switch Layer 3 (multicouche) peut effectuer le routage inter-VLAN directement, sans routeur externe. On crée des SVI (Switch Virtual Interface) pour chaque VLAN.

! Activer le routage IP sur le switch L3
ip routing

! Créer les VLANs
vlan 10
 name COMPTA
vlan 20
 name IT

! Créer les SVI (interfaces VLAN)
interface vlan 10
 ip address 192.168.10.1 255.255.255.0
 no shutdown

interface vlan 20
 ip address 192.168.20.1 255.255.255.0
 no shutdown

ip routing : active le routage sur le switch L3. Sans cette commande, les SVI ne permettent que la gestion, pas le routage inter-VLAN.

Avantages du Switch L3 vs Router-on-a-stick

Switch L3

  • Routage matériel (plus rapide)
  • Pas de goulot d'étranglement sur un seul lien
  • Adapté aux grands réseaux
  • Plus cher

Router-on-a-stick

  • Utilise du matériel existant (routeur + switch L2)
  • Moins cher
  • Limité par la bande passante du lien trunk
  • Adapté aux petits réseaux

8
Points clés à retenir
  • Un VLAN = un domaine de broadcast = un sous-réseau IP
  • Access = 1 VLAN (vers hôtes), Trunk = plusieurs VLANs (entre équipements réseau)
  • L'encapsulation 802.1Q ajoute un tag de 4 octets dans la trame Ethernet
  • Le VLAN natif doit correspondre des deux côtés d'un trunk
  • Ne pas utiliser le VLAN 1 en production (sécurité)
  • Router-on-a-stick : sous-interfaces + encapsulation dot1Q + trunk sur le switch
  • Switch L3 : ip routing + SVI (interface vlan X) + no shutdown
  • Les PCs doivent avoir comme passerelle l'IP de la SVI ou sous-interface de leur VLAN
  • Penser à allowed vlan sur les trunks pour la sécurité
  • Vérifier avec show vlan brief et show interfaces trunk
