SISR RZO

ACL (Access Control Lists)

Filtrage du trafic réseau : ACL standard, étendue et nommée

1
Types d'ACL

Une ACL (Access Control List) est une liste ordonnée de règles qui autorisent (permit) ou refusent (deny) le trafic réseau en fonction de critères définis (adresses IP, protocoles, ports).

TypeNumérosFiltre surPlacement
Standard1-99, 1300-1999Adresse source uniquementProche de la destination
Étendue100-199, 2000-2699Source, destination, protocole, portProche de la source
NomméeNom texteStandard ou étendue (selon déclaration)Selon le type

Règles de placement

ACL Standard → Destination
Ne filtre que la source, donc on la place proche de la destination pour ne pas bloquer le trafic vers d'autres réseaux.

ACL Étendue → Source
Filtre précisément (source + destination + port), donc on la place proche de la source pour économiser la bande passante.

2
ACL Standard numérotée

Syntaxe

access-list <1-99> {permit|deny} <source> <wildcard>
access-list <1-99> {permit|deny} host <IP>
access-list <1-99> {permit|deny} any

Exemple : autoriser le réseau 192.168.1.0/24

! Créer l'ACL
access-list 10 permit 192.168.1.0 0.0.0.255
! deny any implicite en fin d'ACL

! Appliquer sur l'interface (proche de la destination)
interface GigabitEthernet0/0
 ip access-group 10 out

Exemple : bloquer un hôte spécifique

access-list 20 deny host 192.168.1.100
access-list 20 permit any

interface GigabitEthernet0/0
 ip access-group 20 in

Important : sans le permit any en fin d'ACL, le deny any implicite bloquera tout le trafic restant !

3
ACL Standard nommée
ip access-list standard AUTORISER_LAN
 permit 192.168.1.0 0.0.0.255
 deny any

interface GigabitEthernet0/0
 ip access-group AUTORISER_LAN out

Avantage des ACL nommées : noms descriptifs, possibilité de supprimer/modifier des lignes individuelles avec no <numéro de séquence>.

4
ACL Étendue numérotée

Syntaxe

access-list <100-199> {permit|deny} <protocole> <source> <wildcard> <destination> <wildcard> [eq <port>]

Exemple : autoriser HTTP et HTTPS uniquement

access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 80
access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 443
access-list 100 permit udp any any eq 53
access-list 100 deny ip any any

interface GigabitEthernet0/0
 ip access-group 100 in

Exemple : bloquer le ping (ICMP) depuis un réseau

access-list 101 deny icmp 10.0.0.0 0.0.0.255 192.168.1.0 0.0.0.255
access-list 101 permit ip any any

interface GigabitEthernet0/1
 ip access-group 101 in
5
ACL Étendue nommée
ip access-list extended WEB_ONLY
 permit tcp 192.168.1.0 0.0.0.255 any eq 80
 permit tcp 192.168.1.0 0.0.0.255 any eq 443
 permit udp any any eq 53
 deny ip any any

interface GigabitEthernet0/0
 ip access-group WEB_ONLY in

8
Vérification & Dépannage
! Voir toutes les ACL configurées
show access-lists

! Voir les ACL appliquées sur une interface
show ip interface GigabitEthernet0/0

! Voir les ACL en running-config
show running-config | include access-list
show running-config | section ip access-list

! Supprimer une ACL
no access-list 10

! Supprimer une ligne dans une ACL nommée
ip access-list standard AUTORISER_LAN
 no 10
9
Points clés à retenir
  • Il y a un deny any implicite en fin de toute ACL (non visible)
  • L'ordre des règles compte : la première correspondance s'applique, les suivantes sont ignorées
  • Une seule ACL par interface, par direction (in ou out)
  • in = trafic entrant dans l'interface du routeur, out = trafic sortant
  • ACL Standard : proche de la destination
  • ACL Étendue : proche de la source
  • Toujours tester avec show access-lists pour vérifier les compteurs (matches)
  • Penser à autoriser le DNS (port 53/udp) si on autorise HTTP/HTTPS
  • Le mot-clé host équivaut au wildcard 0.0.0.0, any équivaut à 0.0.0.0 255.255.255.255
