SISR RZO

OSPF (Open Shortest Path First)

Protocole de routage Link-State, algorithme de Dijkstra, AD = 110

1
Concepts fondamentaux

OSPF est un protocole de routage dynamique de type Link-State (état de lien). Chaque routeur construit une carte complète du réseau (LSDB) et utilise l'algorithme de Dijkstra (SPF) pour calculer le chemin le plus court vers chaque destination.

CaractéristiqueValeur
TypeLink-State
AlgorithmeDijkstra (SPF)
Distance Administrative110
MétriqueCoût = 100 000 000 / Bande passante (bps)
Multicast224.0.0.5 (all OSPF routers), 224.0.0.6 (DR/BDR)
ProtocoleIP protocole 89
ConvergeRapide (quelques secondes)
HiérarchieZones (Area) — Area 0 = backbone obligatoire

2
Configuration de base

Activer OSPF et annoncer les réseaux

router ospf 1
 router-id 1.1.1.1
 network 192.168.1.0 0.0.0.255 area 0
 network 10.0.0.0 0.0.0.3 area 0
 network 172.16.0.0 0.0.0.255 area 0

router ospf 1 : le « 1 » est l'ID de processus local (pas besoin d'être identique sur tous les routeurs).
router-id : identifiant unique du routeur dans OSPF. Format IP. Choisi manuellement ou = plus haute IP de loopback.

Commande network expliquée

network <adresse-réseau> <wildcard> area <numéro>

Attention : le wildcard dans la commande network sert à identifier quelles interfaces participent à OSPF. Ce n'est pas un masque de sous-réseau. Utilisez le wildcard correspondant au masque de l'interface.

3
Route par défaut & Redistribution
! Créer la route par défaut vers le FAI
ip route 0.0.0.0 0.0.0.0 203.0.113.1

! Redistribuer la route par défaut dans OSPF
router ospf 1
 default-information originate

default-information originate : annonce la route par défaut (0.0.0.0/0) à tous les voisins OSPF. Nécessite qu'une route par défaut existe déjà dans la table de routage.

Variante : default-information originate always annonce la route par défaut même si elle n'existe pas dans la table de routage.

4
Passive Interface
router ospf 1
 passive-interface GigabitEthernet0/0
 ! ou pour tout désactiver puis réactiver sélectivement :
 passive-interface default
 no passive-interface Serial0/0/0

passive-interface empêche l'envoi de paquets Hello OSPF sur une interface (typiquement les interfaces LAN). Le réseau est toujours annoncé, mais aucune adjacence ne se forme sur cette interface.

5
États de voisinage OSPF
ÉtatDescription
DownAucun paquet Hello reçu du voisin
InitHello reçu mais le routeur ne se voit pas dans le Hello du voisin
2-WayCommunication bidirectionnelle établie. Élection DR/BDR sur les réseaux multi-accès
ExStartNégociation du master/slave pour l'échange de LSDB
ExchangeÉchange des DBD (Database Description) — résumés des LSA
LoadingDemande (LSR) et réception (LSU) des LSA manquants
FullLSDB synchronisée. Adjacence complète. État normal de fonctionnement

Résumé : Down → Init → 2-Way → ExStart → Exchange → Loading → Full

6
Élection DR / BDR

Sur un réseau multi-accès (Ethernet), OSPF élit un DR (Designated Router) et un BDR (Backup DR) pour réduire le nombre d'adjacences et les échanges de LSA.

Critères d'élection (par ordre de priorité)

  • Priorité OSPF la plus élevée (par défaut = 1, configurable 0-255). Priorité 0 = ne peut pas être DR/BDR.
  • Router-ID le plus élevé (si priorités égales)
! Forcer un routeur comme DR (priorité élevée)
interface GigabitEthernet0/0
 ip ospf priority 255

! Empêcher un routeur d'être DR/BDR
interface GigabitEthernet0/0
 ip ospf priority 0

Important : l'élection DR/BDR n'est pas préemptive. Si un routeur avec une priorité plus haute rejoint le réseau, il ne remplace pas le DR actuel. Il faut redémarrer le processus OSPF (clear ip ospf process).

7
Vérification & Dépannage
! Voir les voisins OSPF et leur état
show ip ospf neighbor

! Voir les interfaces participant à OSPF
show ip ospf interface brief

! Voir les routes OSPF dans la table de routage
show ip route ospf

! Voir la base de données OSPF (LSDB)
show ip ospf database

! Informations générales OSPF
show ip ospf

! Voir les protocoles de routage actifs
show ip protocols

! Debug des paquets OSPF
debug ip ospf adj
debug ip ospf hello

8
Points clés à retenir
  • OSPF est Link-State, utilise Dijkstra, AD = 110, métrique = coût
  • Le coût est calculé : 100 000 000 / bande passante (bps)
  • Toutes les zones doivent être connectées à l'Area 0 (backbone)
  • router-id doit être unique par routeur (format IP)
  • passive-interface sur les interfaces LAN (pas de Hello vers les PCs)
  • default-information originate pour redistribuer la route par défaut
  • Hello timer = 10s, Dead timer = 40s par défaut sur Ethernet
  • Les timers Hello/Dead doivent correspondre entre voisins
  • DR/BDR uniquement sur réseaux multi-accès (Ethernet), pas sur liens point-à-point
  • Vérifier avec show ip ospf neighbor que l'état est FULL
