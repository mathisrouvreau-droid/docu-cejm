SISR RZO

NAT / PAT (Cisco IOS)

Traduction d'adresses réseau : NAT statique, dynamique et PAT (overload)

1
Concepts fondamentaux

NAT (Network Address Translation) traduit une adresse IP privée en adresse IP publique (et inversement) pour permettre aux hôtes d'un réseau local d'accéder à Internet.

TypeDescription
NAT StatiqueAssociation 1-pour-1 permanente entre une IP privée et une IP publique. Utilisé pour les serveurs accessibles depuis l'extérieur.
NAT DynamiqueAssociation dynamique depuis un pool d'adresses publiques. Les adresses sont attribuées à la demande.
PAT (Port Address Translation)Plusieurs IP privées partagent une seule IP publique grâce aux numéros de port (overload). Le plus courant.

Terminologie NAT

TermeSignification
Inside LocalAdresse IP privée de l'hôte interne (ex : 192.168.1.10)
Inside GlobalAdresse IP publique qui représente l'hôte interne vu depuis Internet (ex : 203.0.113.10)
Outside LocalAdresse IP du serveur distant vue depuis le réseau interne
Outside GlobalAdresse IP publique réelle du serveur distant
2
Configuration NAT Statique

Étape 1 : Définir les interfaces inside/outside

interface GigabitEthernet0/0
 description LAN
 ip address 192.168.1.1 255.255.255.0
 ip nat inside
 no shutdown

interface GigabitEthernet0/1
 description WAN
 ip address 203.0.113.1 255.255.255.0
 ip nat outside
 no shutdown

Étape 2 : Créer la traduction statique

! Associer l'IP privée 192.168.1.10 à l'IP publique 203.0.113.10
ip nat inside source static 192.168.1.10 203.0.113.10

! Autre exemple : NAT statique avec port (port forwarding)
ip nat inside source static tcp 192.168.1.20 80 203.0.113.1 8080

NAT statique avec port : redirige le port 8080 de l'IP publique vers le port 80 du serveur interne. Utile pour héberger plusieurs services sur une seule IP publique.

3
Configuration PAT (Overload)

Méthode 1 : PAT avec l'interface de sortie

! Définir le trafic à traduire (ACL standard)
access-list 1 permit 192.168.1.0 0.0.0.255

! Appliquer le PAT sur l'interface de sortie
ip nat inside source list 1 interface GigabitEthernet0/1 overload

Le mot-clé overload est ce qui active le PAT. Sans lui, c'est du NAT dynamique classique (1-pour-1).

Méthode 2 : PAT avec un pool

ip nat pool POOL_PAT 203.0.113.1 203.0.113.1 netmask 255.255.255.0
access-list 1 permit 192.168.1.0 0.0.0.255

ip nat inside source list 1 pool POOL_PAT overload
4
Configuration NAT Dynamique

Pool d'adresses publiques

! Définir le pool d'IP publiques
ip nat pool MON_POOL 203.0.113.1 203.0.113.14 netmask 255.255.255.240

! Définir le trafic autorisé
access-list 1 permit 192.168.1.0 0.0.0.255

! Associer l'ACL au pool (sans overload = 1-pour-1)
ip nat inside source list 1 pool MON_POOL

Attention : si le pool est épuisé (toutes les IP publiques utilisées), les nouveaux hôtes ne pourront pas accéder à Internet. C'est pourquoi le PAT (overload) est préféré en pratique.

5
Vérification & Dépannage

Commandes de vérification

! Voir la table de traduction NAT
show ip nat translations

! Statistiques NAT (nombre de traductions, hits, misses)
show ip nat statistics

! Effacer toutes les traductions NAT dynamiques
clear ip nat translation *

! Debug en temps réel (attention aux performances)
debug ip nat

6
Points clés à retenir
  • ip nat inside se configure sur l'interface côté LAN (réseau privé)
  • ip nat outside se configure sur l'interface côté WAN (Internet)
  • Le mot-clé overload transforme un NAT dynamique en PAT
  • L'ACL standard utilisée pour le NAT doit correspondre au réseau source à traduire
  • NAT statique est obligatoire pour les serveurs qui doivent être joignables depuis Internet
  • Le PAT est la solution la plus utilisée en entreprise (économise les IP publiques)
  • Les traductions NAT dynamiques ont un timeout (par défaut 24h pour TCP, 60s pour UDP)
  • Vérifier avec show ip nat translations que les traductions apparaissent après un ping
