SISR RZO

RIP (Routing Information Protocol)

Protocole de routage Distance-Vector, métrique = nombre de sauts, AD = 120

1
Concepts fondamentaux

RIP (Routing Information Protocol) est un protocole de routage dynamique de type Distance-Vector (vecteur de distance). Il utilise le nombre de sauts (hop count) comme métrique. Maximum 15 sauts ; 16 = inaccessible.

CaractéristiqueValeur
TypeDistance-Vector
AlgorithmeBellman-Ford
Distance Administrative120
MétriqueHop count (nombre de sauts, max 15)
Mises à jourToutes les 30 secondes (broadcast/multicast)
TransportUDP port 520
ConvergenceLente (minutes)
2
Configuration RIPv2

Configuration de base

router rip
 version 2
 no auto-summary
 network 192.168.1.0
 network 10.0.0.0
 network 172.16.0.0

version 2 : active RIPv2 (multicast, supporte VLSM/CIDR).
no auto-summary : désactive le résumé automatique des routes (essentiel pour les réseaux discontinus).
network : annonce les réseaux classful (pas de wildcard, pas de masque).

Passive interface

router rip
 passive-interface GigabitEthernet0/0
 ! ou
 passive-interface default
 no passive-interface Serial0/0/0

passive-interface empêche l'envoi de mises à jour RIP sur une interface (LAN vers les PCs). Le réseau est toujours annoncé aux autres routeurs.

Route par défaut

ip route 0.0.0.0 0.0.0.0 203.0.113.1

router rip
 default-information originate

RIPv1 vs RIPv2
CaractéristiqueRIPv1RIPv2
Type de diffusionBroadcast (255.255.255.255)Multicast (224.0.0.9)
Classful / ClasslessClassful (pas de masque)Classless (VLSM, CIDR)
Masque de sous-réseauNon transmisTransmis dans les mises à jour
AuthentificationNonOui (texte clair ou MD5)
Résumé automatiqueOui (obligatoire)Oui (désactivable avec no auto-summary)
Réseaux discontinusNon supportésSupportés (avec no auto-summary)

En examen : toujours utiliser RIPv2 avec no auto-summary. RIPv1 est obsolète et ne supporte pas le subnetting moderne (VLSM).

4
Authentification MD5
! Créer le trousseau de clés
key chain RIP_KEYS
 key 1
  key-string MonMotDePasse123

! Appliquer sur l'interface
interface Serial0/0/0
 ip rip authentication mode md5
 ip rip authentication key-chain RIP_KEYS

L'authentification MD5 garantit que les mises à jour RIP proviennent bien d'un routeur autorisé. Le key-chain et le key-string doivent correspondre sur les deux routeurs voisins.

Mécanismes anti-boucle
MécanismeDescription
Split HorizonNe pas renvoyer une route vers l'interface par laquelle elle a été apprise
Route PoisoningAnnoncer une route tombée avec une métrique de 16 (inaccessible)
Poison ReverseRépondre au Route Poisoning en confirmant la métrique 16
Hold-down TimerPendant 180s, ignorer les mises à jour sur une route empoisonnée
Triggered UpdatesEnvoyer une mise à jour immédiate lors d'un changement (sans attendre 30s)

Lecture de show ip route rip

R    192.168.2.0/24 [120/1] via 10.0.0.2, 00:00:15, Serial0/0/0
R    192.168.3.0/24 [120/2] via 10.0.0.2, 00:00:15, Serial0/0/0

R = route RIP. [120/1] = AD 120 / métrique 1 saut. via 10.0.0.2 = prochain saut. 00:00:15 = temps depuis la dernière mise à jour.

7
Points clés à retenir
  • RIP est Distance-Vector, AD = 120, métrique = hop count (max 15)
  • Toujours utiliser version 2 + no auto-summary
  • network utilise des adresses classful (pas de masque, pas de wildcard)
  • Mises à jour toutes les 30 secondes en multicast 224.0.0.9 (v2)
  • Convergence lente (plusieurs minutes) comparé à OSPF
  • passive-interface sur les interfaces LAN
  • RIP ne tient pas compte de la bande passante (un lien 10 Mbps = 1 saut, comme un lien 10 Gbps)
  • Limité à des réseaux de petite taille (max 15 sauts)
  • default-information originate pour redistribuer la route par défaut
  • En entreprise, OSPF est préféré à RIP pour sa convergence rapide et sa scalabilité
